# Workflow

## Order Before Implementing

1. `STATUS.md`
2. `WORKFLOW.md`
3. `CODEX.md`
4. `AGENTS.md`
5. `docs/AI_CONTEXT.md`
6. `docs/TECH_STACK.md`
7. `docs/TASKS.md`
8. `docs/SCENE_MAP.md`
9. `docs/QA_CHECKLIST.md`

## Working Rules

- Work in a feature branch.
- Keep content visible in Spanish.
- Keep component, variable, and function names in English.
- Use `src/data` for professional content.
- Keep phase scope tight.
- Run `npm run lint` and `npm run build` before closing a task.
