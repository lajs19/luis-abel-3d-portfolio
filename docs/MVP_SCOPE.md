# MVP Scope — Luis Abel 3D Portfolio

Este documento define el alcance exacto de la primera versión funcional del portafolio 3D.

El objetivo del MVP no es construir el mundo completo, sino demostrar el concepto con una experiencia clara, profesional, funcional y técnicamente estable.

## Objetivo del MVP

Crear una primera versión funcional del CV/portafolio 3D donde el visitante pueda:

1. Entrar a una escena 3D básica tipo oficina/taller técnico.
2. Ver una presentación profesional de Luis Abel.
3. Interactuar con objetos principales.
4. Abrir paneles informativos.
5. Conocer habilidades técnicas principales.
6. Ver proyectos destacados de forma básica.
7. Descargar el CV.
8. Acceder al contacto profesional.

## Zonas incluidas en el MVP

### Zona 1 — Entrada / Presentación

Estado: MVP obligatorio.

Debe incluir:

- Nombre profesional.
- Título principal de posicionamiento del sitio.
- Subtítulo profesional.
- Resumen corto.
- Botón de descarga de CV.
- Botón o enlace de contacto.

### Zona 2 — Escritorio CAD

Estado: MVP obligatorio.

Debe incluir:

- Software principal.
- Diseño mecánico.
- Modelado 3D.
- Planos técnicos.
- Renderizado.
- Documentación técnica.

### Zona 6 — Pared de proyectos

Estado: MVP básico.

Debe incluir:

- Presa Montegrande.
- Presa Yacahueque.
- Parque Fotovoltaico Bayasol.

Para el MVP se permite mostrar fichas incompletas siempre que los campos no confirmados aparezcan como “Pendiente de completar” o “Información no publicada”.

No inventar responsabilidades, resultados o datos técnicos.

### Zona 10 — Contacto

Estado: MVP obligatorio.

Debe incluir:

- Email profesional.
- LinkedIn.
- Descarga de CV.
- Contacto por WhatsApp si se confirma.
- Portafolio externo si se confirma.

## Zonas fuera del MVP inicial

Estas zonas quedan para Fase 2 o posterior:

- Zona 3 — Diseño arquitectónico/civil completo.
- Zona 4 — Línea de transmisión eléctrica con modelo detallado.
- Zona 5 — Fundaciones con maqueta detallada.
- Zona 7 — Laboratorio completo de simulación.
- Zona 8 — Biblioteca de normas.
- Zona 9 — Certificaciones completas en pared.

En el MVP pueden existir como objetos placeholder no interactivos o deshabilitados, pero no deben requerir desarrollo completo.

## Interactividad del MVP

El MVP debe tener:

- Objetos clicables.
- Hover visual.
- Panel de información reutilizable.
- Cierre de panel.
- Datos cargados desde `src/data`.
- Escena 3D liviana.
- Cámara básica.
- Experiencia responsive mínima.

## No incluido en el MVP

No incluir todavía:

- Modelos 3D pesados.
- Física.
- Sonidos.
- Animaciones complejas.
- Navegación tipo videojuego.
- Multilenguaje.
- Formulario de contacto avanzado.
- CMS.
- Base de datos.
- Login.
- Analytics avanzado.
- Efectos visuales costosos.
- Tour completo con cinemáticas complejas.

## Criterio de éxito

El MVP se considera exitoso si:

- El sitio compila.
- La escena 3D carga.
- Los objetos principales abren paneles.
- El contenido profesional es claro.
- No se inventa información.
- El CV puede descargarse.
- El contacto es visible.
- El sitio funciona de forma aceptable en desktop y móvil.

## Regla principal

Primero claridad. Luego estética. Luego complejidad.
