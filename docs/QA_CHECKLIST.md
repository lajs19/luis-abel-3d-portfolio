# QA Checklist

## Build

* [ ] El proyecto instala dependencias sin errores.
* [ ] `npm run lint` pasa.
* [ ] `npm run build` pasa.
* [ ] No hay errores TypeScript.
* [ ] No hay imports rotos.

## Contenido

* [ ] No se inventa experiencia.
* [ ] No se inventan certificaciones.
* [ ] No se afirma título no completado.
* [ ] La formación aparece como “formación universitaria avanzada”.
* [ ] El contenido visible está en español.
* [ ] El contenido profesional está en `src/data`.

## UX

* [ ] El visitante entiende el concepto en menos de 10 segundos.
* [ ] Los objetos interactivos son claros.
* [ ] Los paneles se pueden cerrar.
* [ ] El contacto es fácil de encontrar.
* [ ] El CV se puede descargar.
* [ ] La navegación no confunde.

## 3D

* [ ] La escena carga correctamente.
* [ ] Los objetos clicables responden.
* [ ] No hay modelos demasiado pesados.
* [ ] Las texturas están optimizadas.
* [ ] La cámara no se pierde.
* [ ] Funciona en desktop.
* [ ] Funciona de forma aceptable en móvil.

## Performance

* [ ] La carga inicial es razonable.
* [ ] Hay lazy loading cuando sea necesario.
* [ ] No hay animaciones innecesariamente costosas.
* [ ] No se usan sombras excesivas.
* [ ] No hay texturas gigantes.
* [ ] No hay modelos sin optimizar.

## Accesibilidad

* [ ] Hay textos alternativos donde aplique.
* [ ] Botones tienen labels claros.
* [ ] Contraste visual suficiente.
* [ ] Los paneles pueden cerrarse fácilmente.
* [ ] La experiencia no depende únicamente del color.
