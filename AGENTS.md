# AGENTS.md

## Instructions for AI Agents

Este archivo aplica para cualquier agente de IA que trabaje en el proyecto: ChatGPT, Claude, Codex, Gemini u otra herramienta similar.

## Archivos que deben leerse antes de trabajar

Antes de modificar código, contenido, diseño o estructura, leer en este orden:

- `STATUS.md` ← leer siempre primero
- `WORKFLOW.md` ← leer siempre segundo
- `README.md`
- `docs/AI_CONTEXT.md`
- `docs/PROJECT_BRIEF.md`
- `docs/SCENE_MAP.md`
- `docs/TASKS.md`
- `docs/ROADMAP.md`

## Reglas generales

- No inventar experiencia profesional.
- No inventar titulos academicos.
- No inventar certificaciones.
- No inventar resultados de proyectos.
- No afirmar que Luis Abel es graduado en Ingenieria Mecatronica.
- Mantener contenido visible en espanol.
- Usar ingles para nombres de componentes, variables, funciones y archivos de codigo.
- Usar TypeScript.
- Mantener los datos profesionales en `src/data`.
- No hardcodear contenido profesional dentro de componentes.
- Mantener componentes reutilizables.
- Optimizar modelos 3D antes de integrarlos.
- Evitar modelos 3D pesados.
- Priorizar rendimiento, accesibilidad y experiencia movil.
- Documentar decisiones importantes en `docs/DECISIONS.md`.
- Documentar errores importantes en `docs/ERROR_LOG.md`.

## Concepto que no debe perderse

Este proyecto no es un CV tradicional ni una pagina web estatica.

Es un CV/portafolio inmersivo en 3D representado como una oficina tecnica, taller CAD y laboratorio de ingenieria.

Cada objeto dentro de la escena debe comunicar una parte del perfil profesional de Luis Abel.

## Prioridad del MVP

Primero debe lograrse:

- Claridad profesional.
- Escena 3D funcional.
- Objetos interactivos basicos.
- Paneles de informacion claros.
- Descarga del CV.
- Enlace de contacto.
- Rendimiento aceptable.

No priorizar animaciones complejas, fisicas, sonido o modelos pesados antes de tener un MVP funcional.
